# IMPROVEMENT LOG

> Non-urgent optimization ideas, tech debt, and "would be nice" changes.
> Separate from `progress/backlog.md` on purpose: backlog.md is committed scope for MVP,
> this file is a parking lot so good ideas aren't lost but also don't derail EXECUTION_MODE.

---

## Rule
- If an improvement idea comes up mid-task, log it here and keep executing the current task.
- Never pull an item from here into active work without moving it to `progress/backlog.md` first
  (that's a scope change — follow `governance/scope.md`'s change protocol).

---

```
| ID   | Idea                         | Category      | Value                | Cost to implement | Status |

```
